# 家校签收通前端开发说明

- **版本**：v0.8
- **日期**：2026-06-07
- **范围**：正式前端工程目录、入口和本地开发约定

## 1. 正式前端目录

正式前端工程位于：

```text
web
```

当前采用 Next/React。旧的 `prototype` 目录已移除；原有可运行页面已迁移到 `web`，内部评审总览页不再作为开发入口保留。

## 2. 页面入口

- `/`：正式使用入口。普通访问进入学校管理员/班主任登录；带 `?t=` 或 `?token=` 的家长签收链接进入家长 token 解析、绑定和签收链路。
- `/ops`：平台运维后台入口。

登录后的学校管理员端和班主任端只使用 `ConsoleLayout` 作为控制台外壳；产品身份、侧边导航、当前账号和退出入口都收在该布局内，不再额外叠加外层品牌顶栏。

## 3. 本地启动

```powershell
cd web
corepack pnpm install
corepack pnpm dev
```

默认前端地址：

```text
http://localhost:3000
```

## 4. 构建验证

```powershell
cd web
corepack pnpm build
```

生产部署采用 Next 静态导出，构建后生成 `web/out`，由 Caddy 直接托管；部署构建时应设置：

```powershell
$env:NEXT_PUBLIC_API_BASE_URL = "https://sign.example.edu.cn"
corepack pnpm build
```

## 5. 后端联调

后端默认地址：

```text
http://localhost:8088
```

前端页面中已接入本地 API 的能力包括：平台运维登录与学校空间管理、学校管理员登录与设置、通知发布（含自定义类型和 PDF 附件）、班级进度、导出归档、班主任后端上下文、班主任今日待办、班主任学生导入、通知工作台、家长链接解析、家长绑定和家长签收。

班主任端顶部学校/班级、群通知文案和本班导出范围都应使用后端返回的当前账号上下文；今日待办由本班通知投递、签收进度和异常状态自动生成，不使用演示待办数据。

批量导入模板只下载表头，不带示例数据行；用户在表头下填写真实数据后上传，避免示例内容被误导入。教师账号导入模板包含可选“初始密码”列，留空时由后端生成。

后端生成家长签收链接时使用 `PUBLIC_APP_BASE_URL`，本地默认是 `http://localhost:3000`。部署时应把它改成你的正式 HTTPS 域名，例如 `https://sign.example.edu.cn`。

## 6. 开发边界

- 前端正式代码放在 `web/`，不要再新增 `prototype/` 目录作为运行时代码入口。
- 组件按正式视图组织在 `web/components/views`。
- 本地演示兜底数据放在 `web/lib/demo-data.ts`，只能使用假数据。
- 新增页面、状态或产品决策后，必须同步更新 `README.md` 或相应公开文档；完整内部 PRD 不随开源仓库发布。
