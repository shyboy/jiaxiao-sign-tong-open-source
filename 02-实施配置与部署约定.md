# 家校签收通实施配置与部署约定

- **版本**：v0.2
- **日期**：2026-06-05
- **用途**：记录 MVP 服务器测试阶段的非敏感部署模板。

## 1. 重要安全约定

本文件只保存非敏感配置，不保存 SSH 私钥、密码、Token、Cookie、数据库密码等秘密信息。

SSH 私钥、服务器密码、Token、Cookie、数据库密码和真实运维账号不得写入仓库、Markdown 文档、脚本、环境变量模板或日志。正式试点前建议使用独立部署用户，不使用 root 长期登录。

## 3. 部署阶段策略

内部流程测试阶段可以先使用：

```text
http://203.0.113.10:{应用端口}
```

该地址只允许用于假数据、脱敏数据或内部演示，不得采集真实学生、家长、签名图片、IP/设备记录。

本次线上部署目标访问地址：

```text
https://sign.example.edu.cn
```

说明：IP + HTTP 端口只适合流程测试和小范围内部演示，不承担正式家长签收。真实数据试点必须通过域名 + HTTPS 访问。可使用 Caddy 或 Nginx 监听 80/443，`/api/*` 反代到后端 `8088`，其余路径托管前端静态导出文件。

## 4. 端口建议

| 服务 | 建议端口 | 说明 |
|---|---:|---|
| 后端 API | `8088` | 部署前先检查端口是否被占用 |
| 前端本地开发 | `3000` | `web/` 目录下的 Next 开发服务默认端口 |
| SSH | `22` | 按服务器安全策略调整 |

如果后端 `8088` 被占用，优先选择相邻未占用端口，例如 `8089`。正式部署时，前端访问地址应同步配置到后端 `PUBLIC_APP_BASE_URL`，用于生成家长签收链接。

## 5. 目录建议

| 用途 | 建议路径 |
|---|---|
| 项目目录 | `/opt/jiaxiao-sign` |
| 前端静态文件 | `/opt/jiaxiao-sign/web-out` |
| 后端工程 | `/opt/jiaxiao-sign/server` |
| 数据目录 | `/opt/jiaxiao-sign/server/data` |
| 上传与签名文件 | `/opt/jiaxiao-sign/server/data/uploads` |
| 通知 PDF 附件 | `/opt/jiaxiao-sign/server/data/uploads/notice-attachments` |
| PDF 导出文件 | `/opt/jiaxiao-sign/server/data/exports` |
| 备份目录 | `/opt/jiaxiao-sign/backups` |
| 环境变量文件 | PM2 环境变量，不写入仓库 |

生产环境至少设置 `PUBLIC_APP_BASE_URL`、`NEXT_PUBLIC_API_BASE_URL`、`CORS_ORIGIN`、`OPS_ADMIN_PASSWORD` 和 `PDF_FONT_PATH`；环境变量值不得提交到仓库。

PDF 导出需要可嵌入的中文 TTF/OTF 字体；Windows 本地会自动尝试常见中文字体。Linux 服务器可安装 `fonts-lxgw-wenkai`、Noto CJK 或文泉驿字体，并通过 `PDF_FONT_PATH` 指定字体路径。

## 6. 备份约定

MVP 测试阶段建议：

- 每日备份数据库和 `data` 目录。
- 通知 PDF 附件、签名图片、导出文件都属于 `data` 目录数据；PDF 附件只能通过后端鉴权或 token 接口访问，不能把上传目录配置成公网静态目录。
- 备份保留 7 天。
- 备份文件命名格式：`jiaxiao-sign-YYYYMMDD-HHmmss.tar.gz`。

真实班级扩大试点前，需要再确认备份频率、保留周期和恢复演练方式。

## 7. 管理员账号命名建议

| 账号类型 | 命名建议 |
|---|---|
| 平台维护账号 | `platform_admin` |
| 学校管理员账号 | `school_admin_{学校简称}` |
| 班主任账号 | `teacher_{班级简称或工号}` |

初始密码由学校创建并通过老师群或指定工作群告知。MVP 不强制学校管理员和班主任首次登录修改初始密码；如学校内部要求，可通过登录后的密码修改接口手动更换。

## 8. 本地 demo 数据约定

后端默认不自动写入学校 demo 数据，避免流程测试时混入示例学校、示例通知、示例学生和签收记录。

如确需恢复本地演示学校数据，可在启动后端前设置：

```powershell
$env:JIAXIAO_SEED_DEMO = "1"
npm run start
```

平台运维账号用于本地流程测试，学校管理员和班主任账号应通过 `/ops` 开通学校后创建。后端仅在 `ops_users` 为空时创建平台运维账号；用户名由 `OPS_ADMIN_USERNAME` 指定，默认 `platform_admin`；密码由 `OPS_ADMIN_PASSWORD` 指定，未指定时首次启动会生成一次性随机密码并输出到启动日志。已有运维账号时，启动逻辑不得覆盖既有运维密码。
